## The construction (Section 3)

**Monotonicity indicator vector** `t ∈ {−1, 0, 1}ⁿ` (Eq. 1): `+1` if the
output is non-decreasing in input `xⱼ`, `−1` if non-increasing, `0` if
unconstrained (non-monotone).

**Weight-sign operator `|·|_t`** (Eq. 2): given matrix `M` and indicator `t`,

```
(|M|_t)_{j,i} =  |m_{j,i}|   if t_i = +1
              = −|m_{j,i}|   if t_i = −1
              =  m_{j,i}     otherwise (t_i = 0, unconstrained)
```

**Constrained linear layer** (Def. 1, Eq. 3): `h = |Wᵀ|_t · x + b`. Lemma 1:
`∂hⱼ/∂xᵢ ≥ 0` when `tᵢ=+1`, `≤ 0` when `tᵢ=−1`.

**Activation classes.** `Ă` = zero-centred, monotone-increasing, convex,
lower-bounded functions (ReLU, ELU, SELU, …). From a base `ρ̆ ∈ Ă` build
two siblings (Def. 2):

```
ρ̂(x) = −ρ̆(−x)                                  (Eq. 4)  concave reflection
ρ̃(x) = ρ̆(x+1) − ρ̆(1)   if x < 0               (Eq. 5)  saturated / bounded
      = ρ̂(x−1) + ρ̆(1)   otherwise
```

**Combined activation** `ρ^s` (Def. 3, Eq. 6) with split
`s = (s̆, ŝ, s̃) ∈ ℕ³`, `s̆ + ŝ + s̃ = m`: the first `s̆` neurons get `ρ̆`
(convex), the next `ŝ` get `ρ̂` (concave), the rest get `ρ̃` (saturated).

**CMFCL / Monotonic Dense Unit** (Def. 4, Eq. 7):

```
y = ρ^s( |Wᵀ|_t · x + b )
```

Corollary 3: `y` is non-decreasing in `+1` inputs, non-increasing in `−1`
inputs; `s=(m,0,0)` ⇒ convex, `s=(0,m,0)` ⇒ concave. The **saturated `ρ̃`
class is what makes the construction a universal approximator** (not just
convex/concave functions) — needed e.g. to fit a sigmoid.

## Architectures (Section 3.2)

- **Type 1 (Fig. 4) — MLP.** Concatenate all inputs into `x`; the input layer
  uses the user's indicator `t`; **all deeper layers fix `t = 1`** to preserve
  monotonicity. Task-appropriate output activation (linear/sigmoid/softmax).
  This is `mononet`'s `MonoMLP`.
- **Type 2 (Fig. 5) — feature block.** Non-monotone features go through an
  arbitrary external extractor; each monotone input gets its own monotonic
  dense unit (so `s`/`t` can be tuned per feature); feature vectors are then
  concatenated and fed to a monotone tail.

## Universal approximation (Section 3.3)

Builds on Daniels & Velikova 2010 (Theorem 4: a constrained sigmoid network
with ≤ `k` layers approximates any monotone continuous function on a compact
`K ⊆ ℝᵏ`). Lemma 5: the Heaviside function is approximable by `ρ̃_H(x) =
αρ̃(x) + β`, so the saturated activation lets the construction inherit the
universal property for *any* `ρ̆ ∈ Ă`, not just bounded sigmoids.

## How this maps onto `mononet`'s design

- `mode="absolute"` is this paper's `|W|_t` construction. **Design deviation
  (decided for this package):** rather than the three-class split
  `(s̆, ŝ, s̃)`, `mononet` uses a **two-class** convex/concave split (`ρ̆` and
  `ρ̂` only, default half/half), dropping the saturated `ρ̃`. This is justified
  by Sartor et al. 2025 Prop. 3.9: alternating a convex activation and its
  point reflection across depth is already a universal monotonic approximator,
  so `ρ̃` is not required. See
  [Sartor digest](2505.02537-sartor-2025-advancing-cmnn.md).
- The indicator `t` does **not** live on the `Mono` layer in `mononet`; the
  layer is non-decreasing in all inputs. Direction (`±1`) is applied once, at
  `MonoMLP`'s input, via input-sign flips. Partial monotonicity (`t=0`) is
  handled by the user outside the library (external net for non-monotone
  features), so `mononet` does not ship the Type-2 block.
